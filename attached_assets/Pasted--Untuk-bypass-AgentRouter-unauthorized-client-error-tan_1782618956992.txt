**Untuk bypass AgentRouter "unauthorized_client_error" tanpa multiple accounts (tanpa banyak key baru), fokusnya adalah mimic fingerprint/official client "wire image" sebaik mungkin di proxy layer kamu.**

### Root Cause Utama
AgentRouter (agentrouter.org) pakai **client fingerprinting ketat** (bukan cuma User-Agent atau IP). Mereka cek pola request yang mirip official SDK (OpenAI/Anthropic) atau "Claude Code wire image" (specific headers, order, TLS fingerprint/JA3, connection behavior, dll). Server-side / custom proxy sering gagal match, sehingga setelah 1 request sukses, key di-flag sebagai "unauthorized client" sementara.

Cloudflare Worker kamu sudah bagus bypass WAF (karena residential-like atau clean IP), tapi fingerprint ke AgentRouter masih kurang.

### Solusi Terbaik Tanpa Multiple Accounts
1. **Enhance Fingerprint di Cloudflare Worker (Rekomendasi Utama)**
   - Jangan cuma spoof User-Agent. Tambahkan **full header set** dari official client.
   - Gunakan library seperti `undici` atau fetch dengan custom context di Worker untuk control TLS fingerprint (susah di CF, tapi bisa di-approximate).
   - Mimic **Claude Code / Anthropic SDK** pattern (banyak issue GitHub yang bahas ini).

   Contoh header penting yang sering dibutuhkan (dari komunitas & wiki):
   - `User-Agent`: Sesuatu seperti official OpenAI/Anthropic SDK (e.g., `OpenAI/Node.js/4.x` atau Anthropic equivalent).
   - `Accept`: `application/json`
   - `Content-Type`: `application/json`
   - `Authorization`: `Bearer sk-...`
   - Tambahan headers seperti `anthropic-version`, `x-api-key` (kalau Anthropic mode), atau OpenAI-specific.
   - Order headers konsisten (penting untuk fingerprint).
   - Hindari extra headers yang mencurigakan.

   Di `v1-proxy.ts` kamu, pastikan proxy **forward request body & headers hampir identik**, hanya inject/replace base URL + key rotation.

2. **Key Rotation + Cooldown yang Lebih Smart**
   - Kamu sudah punya ini → bagus. Isi `AGENTROUTER_API_KEY` dengan 5-10 keys (pisah koma).
   - Implement round-robin + cooldown per key (misal 5-15 menit block per key).
   - Rotate **sebelum** error muncul, bukan setelah. Track success rate per key.

3. **Alternative Proxies / Chains (tanpa new account)**
   - **Worker → Another Residential Proxy** (e.g., BrightData, Oxylabs, atau self-hosted proxy di VPS residential) sebelum ke AgentRouter. Ini ubah IP + TLS fingerprint.
   - Gunakan **official SDK di client-side** (bukan pure server proxy) kalau memungkinkan, lalu forward hanya hasilnya.
   - Test dengan **unofficial SDK** seperti `@rxmxdhxni/agentrouter` yang diklaim handle fingerprint.

4. **Lainnya yang Bisa Dicoba**
   - **TLS Fingerprint Spoofing**: Pakai tools seperti `curl-impersonate` atau Go dengan uTLS di backend (kalau Worker kurang, deploy simple Go proxy di Fly.io / Railway dengan residential IP).
   - **Request Pattern**: Kirim request persis seperti browser atau official client (termasuk streaming handling kalau relevan).
   - Cek dashboard AgentRouter untuk melihat limit/fingerprint status per key.

### Kode Snippet Ide untuk Worker (`v1-proxy.ts`)
Pastikan proxy seperti ini (pseudocode):

```ts
// Di handler request
const headers = new Headers(request.headers);
headers.set('User-Agent', 'Official SDK UA'); // ganti dengan exact
// hapus / override suspicious headers
headers.delete('cf-ray'); // dll

const response = await fetch(`https://agentrouter.org${url.pathname}`, {
  method: request.method,
  headers,
  body: request.body,
  // redirect: 'manual', dll
});
```

Test dengan tool seperti `curl -v` atau Postman dengan full headers dari successful browser request.
