Yang rentan ⚠️
1. DDoS — tidak ada perlindungan layer apapun Server langsung kena request tanpa filter. Route seperti GET /api/projects, POST /api/deploy, dll tidak punya rate limit sama sekali. Satu script bisa flood server sampai mati.

2. Rate limit hanya di auth, bukan global Semua route lain (billing, projects, deployments) bisa di-spam bebas.

3. Rate limit pakai memory — tidak tahan restart Kalau server restart, counter reset. Attacker cukup tunggu deploy.

4. Belum ada CSP header Celah kecil untuk XSS kalau ada third-party script.

Prioritas perbaikan
Yang harus pakai infrastruktur (bukan kode):

Cloudflare — ini satu-satunya cara efektif lawan DDoS volume besar. Free tier sudah cukup. Aktifkan Proxy mode, otomatis dapat: DDoS mitigation L3/L4/L7, rate limiting, IP reputation filter, hide origin IP. Tanpa ini, sekuat apapun kodenya, server tetap bisa diflood langsung.
