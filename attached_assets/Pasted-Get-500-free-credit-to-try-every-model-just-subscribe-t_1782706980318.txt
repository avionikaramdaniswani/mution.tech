The model can't touch your computer
A common worry: that connecting our models to Claude Code, Codex or other agents lets the AI run commands, download malware or plant a RAT on your machine. It can't. Here is exactly how it works, in plain terms

It only returns text
Our API is a text service. It sends your request to the model and returns the answer as text — nothing more. It has no access to your files, your shell or your network. It physically cannot download or run anything on your computer.

Agents ask before acting
Tools like Claude Code and Codex run on your machine, not ours. They show you every command and every file change and wait for your approval. Nothing runs unless you allow it — that gate belongs to the tool you installed, and Conduit never removes it.

You stay in control
The model proposes; you decide. Read each suggested command, deny anything you don't like, or run the agent read-only or inside a sandbox or VM. Using Conduit instead of a provider's own key changes none of these local safeguards.

Step 1
You send a request
From the agent on your computer
Step 2
Conduit returns text
An answer or a suggested command — never an action
Step 3
The agent asks you
Approve, deny or edit every command and file change
Step 4
Only then it runs
On your machine, with your explicit yes
In short: an AI model is not a program running on your PC — it's a remote text service. The scenario people fear, a model silently installing malware through the API, isn't technically possible. The only thing that ever runs locally is the agent tool you chose to install, and that tool always asks you first. We never execute your code, never store your files and have no agent on your side.

Quick start
Conduit speaks the OpenAI Chat Completions format. If your code already talks to OpenAI, change two things: the base URL and your key. Everything else stays the same

Base URL: https://conduit.ozdoev.net/api/v1
Auth header: Authorization: Bearer sk-cdt-…
Anthropic base URL (Claude Code, Anthropic SDK): https://conduit.ozdoev.net
Get your key in the Dashboard. Every account starts with $500 of free trial credit — no card needed
Send a message
POST
/api/v1/chat/completions
Send a list of messages and pick any model by its id. Example with curl:

curl https://conduit.ozdoev.net/api/v1/chat/completions \
  -H "Authorization: Bearer sk-cdt-YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-opus-4-8",
    "messages": [
      { "role": "user", "content": "Hello, who are you?" }
    ]
  }'
Response
{
  "id": "chatcmpl-...",
  "object": "chat.completion",
  "model": "claude-opus-4-8",
  "choices": [
    {
      "index": 0,
      "message": { "role": "assistant", "content": "Hi! I'm Claude Opus 4.8." },
      "finish_reason": "stop"
    }
  ],
  "usage": { "prompt_tokens": 12, "completion_tokens": 14, "total_tokens": 26 }
}
Model fidelity
The requested model id is treated as a contract: successful responses echo the same public model id, unknown model ids return 400 instead of silently falling back, and internal routing details are never exposed to clients.

JavaScript (OpenAI SDK)
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: "sk-cdt-YOUR_KEY",
  baseURL: "https://conduit.ozdoev.net/api/v1",
});

const res = await client.chat.completions.create({
  model: "gpt-5",
  messages: [{ role: "user", content: "Write a haiku about APIs." }],
});

console.log(res.choices[0].message.content);

Connect your tools
Conduit exposes two compatible endpoint families — OpenAI-compatible routes on /v1 or /api/v1, and Anthropic Messages on /v1/messages. For Claude Code, use the root base URL, not /api.

Claude Code
First install it (needs Node.js 18+):

npm install -g @anthropic-ai/claude-code
Then point it at Conduit's Anthropic endpoint and run it:

Linux / macOS (bash / zsh)

unset ANTHROPIC_AUTH_TOKEN
export ANTHROPIC_BASE_URL="https://conduit.ozdoev.net"
export ANTHROPIC_API_KEY="sk-cdt-YOUR_KEY"
export ANTHROPIC_MODEL="claude-opus-4-8"
export CLAUDE_CODE_SKIP_FAST_MODE_ORG_CHECK="1"
claude
Windows (PowerShell)

Remove-Item Env:ANTHROPIC_AUTH_TOKEN -ErrorAction SilentlyContinue
$env:ANTHROPIC_BASE_URL="https://conduit.ozdoev.net"
$env:ANTHROPIC_API_KEY="sk-cdt-YOUR_KEY"
$env:ANTHROPIC_MODEL="claude-opus-4-8"
$env:CLAUDE_CODE_SKIP_FAST_MODE_ORG_CHECK="1"
claude
Windows (CMD)

set ANTHROPIC_AUTH_TOKEN=
set ANTHROPIC_BASE_URL=https://conduit.ozdoev.net
set ANTHROPIC_API_KEY=sk-cdt-YOUR_KEY
set ANTHROPIC_MODEL=claude-opus-4-8
set CLAUDE_CODE_SKIP_FAST_MODE_ORG_CHECK=1

Tool / function calling
Full OpenAI and Anthropic tool calling is supported — pass tools and tool_choice exactly as you would with the official APIs; tool_calls come back in the standard shape. When you send no tools of your own, the model can search the web on its own.

Thinking / reasoning mode
Models that officially support extended thinking (Claude Opus 4.8, Sonnet 4.6, Haiku 4.5, GPT-5, GPT-5 mini, o4, Gemini 3 Pro, Gemini 2.5 Pro/Flash, Grok 4, Grok 3 mini, DeepSeek V3.2, DeepSeek R1, Qwen3 Max, Qwen3 Coder) can return a visible reasoning trace before the answer. The thinking stays fully in the model's own voice and never exposes any prompt.

OpenAI route — set reasoning_effort:

curl https://conduit.ozdoev.net/api/v1/chat/completions \
  -H "Authorization: Bearer sk-cdt-YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5",
    "reasoning_effort": "high",
    "messages": [{ "role": "user", "content": "Prove sqrt(2) is irrational." }]
  }'
# reasoning comes back in choices[0].message.reasoning_content
# (and as delta.reasoning_content when stream:true)
Anthropic route — enable thinking:

curl https://conduit.ozdoev.net/v1/messages \
  -H "x-api-key: sk-cdt-YOUR_KEY" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-opus-4-8",
    "max_tokens": 2048,
    "thinking": { "type": "enabled", "budget_tokens": 4096 },
    "messages": [{ "role": "user", "content": "Prove sqrt(2) is irrational." }]
  }'
# a "thinking" content block precedes the "text" block in the response
Parameters
model - model id, e.g. claude-opus-4-8, gpt-5, gemini-3-pro, grok-4
messages - array of { role, content }; role is system, user or assistant
temperature - 0-2, creativity (optional)
max_tokens - cap on the reply length (optional)
stream - true for token streaming (optional)
Errors
401 - missing or invalid key
402 - free trial balance spent; upgrade to ULTIMATE for unlimited
403 - account blocked
429 - fair-use throttling; slow down and retry
